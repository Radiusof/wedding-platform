Project Overview

Wedding Platform is a modern, microservices-based web application designed to provide comprehensive wedding planning and guest management capabilities. Built with scalability and maintainability in mind, it leverages industry-standard technologies and patterns.

Architecture Pattern: Event-driven microservices with Redis pub/sub messaging and RESTful APIs for external communication.
Component Technology Purpose Status
Backend Services NestJS + TypeScript Microservices implementation In Development
Frontend Vue.js 3 + TypeScript User interface Planned
Database MongoDB Data persistence Ready
Message Broker Redis Inter-service communication Ready
Infrastructure Docker + GCP Container orchestration Configured

System Architecture

The platform follows a distributed microservices architecture with clear separation of concerns and standardized communication patterns.

Core Principles

  • Single Responsibility: Each service handles one business domain
  • Event-Driven: Services communicate via Redis pub/sub events
  • API Gateway Pattern: Centralized entry point for all client requests
  • Database per Service: Each service owns its data
Client → API Gateway → [Microservices] ↔ Redis Pub/Sub ↓ MongoDB Collections

Database Schema

MongoDB collections organized by service domain with clear data ownership patterns.

wedding_db/ ├── users/ # User accounts and profiles ├── faqs/ # Questions and answers ├── locations/ # Venue information ├── accommodations/ # Hotel and lodging data ├── playlist/ # Music playlists ├── songs/ # Individual song entries └── notifications/ # User notifications
Collection Service Owner Key Fields Indexes
users User Service email, firstName, lastName, role email (unique), role
faqs FAQ Service question, answer, category, order category, order
locations Location Service name, address, coordinates, type type, coordinates (2dsphere)
accommodations Accommodation Service name, price, rating, distance price, rating, distance

Inter-Service Communication

Services communicate through Redis pub/sub channels with standardized event formats.

Redis Channels

user_events → User lifecycle events faq_events → FAQ content changes location_events → Venue updates accommodation_events → Lodging data changes playlist_events → Music-related events notification_events → Messaging triggers

Event Structure

{ "eventType": "user.created", "timestamp": "2024-01-15T10:30:00Z", "userId": "user123", "data": { "userId": "user123", "user": { ... } } }

Deployment Architecture

Production deployment on Google Cloud Platform with container orchestration.

Development Environment

# Local development stack docker-compose up -d # Start all services docker-compose logs -f # View logs docker-compose restart <service> # Restart specific service

Production Environment (GCP)

  • Cloud Run: Auto-scaling containers for each microservice
  • MongoDB Atlas: Managed database service
  • Redis Cloud: Managed Redis instance
  • Cloud Load Balancer: Traffic distribution
  • Cloud Storage: Static assets and media files
